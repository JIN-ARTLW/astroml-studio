# Quickstart & Validation: AstroML Studio v0

이 기능이 end-to-end로 동작함을 증명하는 실행/검증 가이드. (구현 세부는 tasks.md 및 구현 단계에서.)

## 사전 조건

- Node 20 LTS, 최신 브라우저(WebGL 지원).
- 로컬 실행: `npm install` → `npm run dev` → `http://localhost:3000`.
- 데모 데이터(`public/demo/`, 은하 3클래스)가 번들되어 있어야 함.

## 검증 시나리오

### S1 — 핵심 동선 완주 (P1, SC-001/002/004)
1. 홈 → "새 프로젝트" → 작업 **분류** 선택, 클래스 3개 정의.
2. 데이터 화면 → **"데모 불러오기"**.
3. (데모는 자동 라벨) 라벨링 진행률 100% 확인.
4. 학습 화면 → 백본 MobileNetV2 → **학습 시작** → 에폭별 정확도/손실 라이브 갱신.
5. 결과 화면 → 혼동행렬·클래스별 지표·학습곡선·오분류 갤러리 표시.
6. 예측 화면 → 새 이미지 드롭 → 클래스+신뢰도(top-k).
- **기대**: 코드/설치 없이 완주, 전 과정 ≤ 15분.

### S2 — 정확도 임계 (H2, SC-003)
- S1의 검증 정확도 **≥ 0.80**.

### S3 — 재방문 영속성 (SC-010)
- S1 후 탭을 닫았다 다시 열기 → 프로젝트·이미지·라벨·결과가 **유지**됨.

### S4 — 폴더 업로드 자동 라벨 (FR-005)
- 클래스명 폴더 구조로 업로드 → 폴더명=클래스 자동 매핑, 라벨링 생략.

### S5 — 오류 격리 (FR-006, SC-008)
- 손상/미지원 파일 섞어 업로드 → 해당 항목만 스킵+사유, 나머지 정상 진행.

### S6 — 규모 상한 안내 (FR-029, SC-009)
- 총 ~1,500장 초과 데이터 → 외부 GPU 본격 학습 권유 배너 표시.

### S7 — 노트북 핸드오프 (P2, FR-016~019, SC-006)
1. "본격 학습 노트북 생성" → `.ipynb` 다운로드.
2. 외부 GPU에서 실행 → `result.json` 산출.
3. 업로드 → 브라우저 결과와 **동일 형식**으로 비교 렌더.
- 설정 불일치(클래스 수 등) `result.json` 업로드 → **거부+사유**.

### S8 — 작업 확장 가능성 (SC-007)
- (확장 시) 회귀 등 신규 `TaskPlugin` 등록만으로 동일 6화면 흐름 동작, 코어 변경 0.
