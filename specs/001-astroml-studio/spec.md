## Overview

천문·우주과학 이미지에서 **머신러닝 지식 없이** 원하는 결과(분류·검출·물리량·이상)를 얻게 해주는 웹 기반 노코드 연구툴. 대상은 천문/물리 **학부 고학년 ~ 대학원생**(ML 비전공). 핵심 통찰: "연구자가 원하는 건 ML 모델 자체가 아니라 *이미지에서 알고 싶은 것*"이며, 천문 이미지 ML의 대부분은 정형 파이프라인(전처리→라벨→학습→평가)으로 수렴한다. 본 서비스는 그 파이프라인을 **작업(task) 단위로 추상화**하고, 무료로 쓸 수 있는 **하이브리드 학습 경험**(브라우저 즉시 학습 + 외부 무료 GPU 본격 학습)을 제공한다.

스코프 원칙: **v0는 "분류" 작업을 end-to-end로 깊게**(P1) 구현하고, 나머지 작업·천문 입력·과학적 리포트는 동일 추상화 위에서 단계적으로 확장한다.

## Clarifications

### Session 2026-06-25

- Q: v0에서 프로젝트·이미지·라벨을 어디에 저장할까? → A: **브라우저 로컬 우선**(백엔드 없이 브라우저 로컬 저장. 클라우드 저장·공유는 이후 단계).
- Q: 브라우저(빠른 경로) 학습이 감당할 데이터 규모 상한은? → A: **총 ~1,500장(클래스당 300~500)**. 초과 시 외부 GPU 본격 학습으로 안내.
- Q: UI 언어 범위는? → A: **한국어 우선, 영어 후속**(v0 한국어, 영어는 다음 단계).

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 코드 없이 이미지 분류기 만들기 (Priority: P1)

ML을 모르는 학부생이 자신의(또는 데모) 천문 이미지를 업로드/불러와 클래스를 정의하고, 라벨을 붙인 뒤, 브라우저에서 바로 학습해 분류 결과(정확도·혼동행렬·오분류 사례)를 확인하고, 새 이미지를 예측한다. 설치도 코드도 없다.

**Why this priority**: 이것이 제품의 핵심 가치 가설(H1·H2)을 단독으로 증명하는 최소 가치 단위(MVP). 이 스토리만 구현해도 "ML 몰라도 천문 이미지로 결과를 얻는다"가 성립한다.

**Independent Test**: 데모 데이터셋(갤럭시 형태 3클래스)으로 신규 사용자가 도움말 없이 프로젝트 생성→학습→예측까지 완주할 수 있고, 검증셋 정확도와 혼동행렬이 화면에 표시되면 통과.

**Acceptance Scenarios**:

1. **Given** 처음 방문한 사용자, **When** "분류" 작업으로 새 프로젝트를 만들고 클래스를 정의한 뒤 데모 데이터를 불러오면, **Then** 라벨이 부여된 이미지 갤러리가 표시된다.
2. **Given** 라벨이 부여된 데이터셋, **When** "학습 시작"을 누르면, **Then** 진행 중 에폭별 정확도/손실이 실시간으로 갱신되고 완료 시 검증 정확도가 표시된다.
3. **Given** 학습이 완료된 모델, **When** 결과 화면을 열면, **Then** 혼동행렬·클래스별 정밀도/재현율·오분류 이미지 갤러리가 표시된다.
4. **Given** 학습된 모델, **When** 새 이미지를 올리면, **Then** 예측 클래스와 신뢰도(top-k)가 표시된다.
5. **Given** 직접 업로드한 폴더(폴더명=클래스), **When** 업로드하면, **Then** 폴더명이 클래스로 자동 매핑되어 라벨링 단계가 생략된다.

---

### User Story 2 - 외부 무료 GPU로 본격 학습 핸드오프 (Priority: P2)

브라우저 학습으로 감을 잡은 사용자가, 동일한 데이터셋·라벨 설정으로 **본격 학습용 노트북을 자동 생성**받아 자신의 무료 클라우드 GPU 환경에서 실행하고, 그 결과(모델·평가지표)를 다시 서비스에 업로드해 동일한 결과 화면에서 비교한다.

**Why this priority**: 무료 운영(H4)을 지키면서도 "진짜 연구급 학습"을 가능하게 하는 차별 동선. 브라우저 학습의 규모 한계를 우회한다.

**Independent Test**: 한 프로젝트에서 "본격 학습 노트북 생성"을 실행해 받은 노트북이 외부 GPU 환경에서 그대로 실행되어 표준 결과 파일을 산출하고, 그 파일을 업로드하면 브라우저 학습과 동일한 형식의 결과 화면이 렌더되면 통과.

**Acceptance Scenarios**:

1. **Given** 라벨이 완료된 프로젝트, **When** "본격 학습 노트북 생성"을 선택하면, **Then** 해당 데이터셋·클래스·설정이 반영된 실행 가능한 노트북이 생성/다운로드된다.
2. **Given** 외부 GPU에서 실행해 얻은 결과 파일, **When** 서비스에 업로드하면, **Then** 브라우저 학습과 동일한 표준 지표 화면(정확도·혼동행렬 등)으로 표시된다.
3. **Given** 브라우저 결과와 외부 학습 결과가 모두 존재할 때, **When** 결과를 비교하면, **Then** 두 실행의 지표가 나란히 비교된다.

---

### User Story 3 - 천문 원본 데이터(FITS) 입력과 전처리 (Priority: P3)

사용자가 일반 이미지가 아닌 **천문 표준 포맷(FITS)** 데이터를 올리고, 천문 특화 전처리(배경 제거, 소스 추출, 비선형 stretch 등)를 노코드로 적용한 뒤 동일한 학습 파이프라인에 투입한다.

**Why this priority**: 범용 노코드 ML 툴과의 **결정적 차별점**(천문 도메인 인식). 단, 핵심 가치 증명(P1) 이후여도 무방하므로 P3.

**Independent Test**: FITS 파일을 업로드해 좌표/메타가 인식되고, 전처리 프리뷰가 적용된 이미지로 분류 학습까지 이어지면 통과.

**Acceptance Scenarios**:

1. **Given** FITS 파일, **When** 업로드하면, **Then** 이미지와 함께 천문 메타(좌표계 등)가 인식되어 표시된다.
2. **Given** 업로드된 천문 이미지, **When** 전처리(배경 제거·stretch)를 적용하면, **Then** 적용 전/후 프리뷰가 표시되고 학습 입력으로 사용된다.

---

### User Story 4 - 작업 유형 확장(검출·회귀·이상탐지) (Priority: P4)

사용자가 분류 외에 소스 검출/세그멘테이션, 물리량 회귀(예: photometric redshift), 이상탐지를 **같은 방식의 흐름**으로 수행한다(작업에 맞는 라벨링 UI·평가지표가 제공됨).

**Why this priority**: "한 툴에서 다 된다"는 비전 완성(H3 검증)이지만, 추상화가 P1에서 검증된 뒤 확장하는 것이 안전하다.

**Independent Test**: 분류 외 최소 1개 작업(예: 회귀)을 동일한 프로젝트 흐름으로 끝까지 수행할 수 있고, 그 작업에 맞는 지표(예: RMSE)가 표시되면 통과.

**Acceptance Scenarios**:

1. **Given** 새 프로젝트, **When** 작업 유형으로 "회귀"를 선택하면, **Then** 스칼라값 라벨링 UI와 회귀용 지표(RMSE/MAE)가 제공된다.
2. **Given** 검출 작업, **When** 이미지에 박스를 그려 라벨링하면, **Then** 검출용 지표(mAP)로 결과가 평가된다.

---

### User Story 5 - 과학적 신뢰성 리포트 (Priority: P5)

연구·교육용 신뢰성을 위해, 학습 결과에 **재현성 정보(설정·데이터 분할·버전)** 와 **불확실성/주의(데이터 누수 경고 등)** 가 포함된 리포트가 제공된다.

**Why this priority**: 연구툴로서 장기 신뢰성을 좌우하지만, 핵심 동선이 안정된 후 도입해도 되는 마감 단계 가치.

**Independent Test**: 임의 학습 결과에서 동일 설정으로 재현에 필요한 정보가 모두 리포트에 포함되고, 명백한 데이터 누수 상황에서 경고가 표시되면 통과.

**Acceptance Scenarios**:

1. **Given** 완료된 학습, **When** 리포트를 열면, **Then** 데이터 분할·하이퍼파라미터·데이터셋 식별 정보가 재현 가능하게 기록되어 있다.
2. **Given** train/test에 동일 이미지가 섞인 설정, **When** 학습을 시도하면, **Then** 데이터 누수 경고가 표시된다.

---

### Edge Cases

- 클래스당 표본이 너무 적거나(예: 한 클래스 5장 미만) 클래스 불균형이 심할 때 → 경고 및 권장 최소치 안내.
- 라벨이 전혀 없는 상태에서 학습을 시도할 때 → 학습 차단 및 안내.
- 손상되었거나 지원하지 않는 파일/포맷 업로드 → 개별 항목 스킵 + 사유 표시(전체 실패 금지).
- 브라우저 학습 권장 상한(총 ~1,500장)을 초과하는 데이터셋 → 표본 수 제한 안내 또는 외부 GPU 본격 학습(P2) 경로 권유.
- 브라우저 로컬 저장 용량 한도 근접/초과 → 한도 안내 및 정리/내보내기 경로 제안.
- 외부 GPU 결과 업로드 시 프로젝트 설정과 불일치(클래스 수 등) → 업로드 거부 + 불일치 항목 표시.
- 학습 도중 탭/네트워크 중단 → 진행 상태 보존 또는 명확한 재시작 안내.

## Requirements *(mandatory)*

### Functional Requirements

**프로젝트 & 작업**
- **FR-001**: 시스템은 사용자가 "작업 유형"을 선택해 프로젝트를 생성할 수 있어야 한다. v0에서 활성 작업은 **분류**이며, 그 외 작업은 "준비 중"으로 표시된다.
- **FR-002**: 분류 프로젝트에서 사용자는 클래스(이름) 집합을 정의·수정할 수 있어야 한다.
- **FR-003**: 시스템은 모든 작업 유형을 **동일한 단계 흐름**(데이터→라벨링→학습→결과→예측)으로 일관되게 제공해야 한다.

**데이터 입력**
- **FR-004**: 사용자는 표준 이미지(PNG/JPG)를 업로드하거나, 제공되는 **데모 데이터셋**을 한 번에 불러올 수 있어야 한다.
- **FR-005**: 폴더 단위 업로드 시 시스템은 폴더명을 클래스로 자동 매핑할 수 있어야 한다.
- **FR-006**: 시스템은 지원하지 않거나 손상된 항목을 개별적으로 건너뛰고 사유를 표시해야 하며, 일부 오류가 전체 업로드를 실패시키지 않아야 한다.
- **FR-007**(P3): 시스템은 천문 표준 포맷(FITS) 업로드를 지원하고, 이미지와 함께 천문 메타(좌표계 등)를 인식해야 한다.
- **FR-008**(P3): 시스템은 천문 특화 전처리(배경 제거, 소스 추출, 비선형 stretch 등)를 노코드로 적용하고 적용 전/후 프리뷰를 제공해야 한다.

**라벨링**
- **FR-009**: 사용자는 갤러리에서 이미지에 라벨을 부여할 수 있어야 하며, 분류는 클래스 단일 선택과 키보드 단축키를 지원해야 한다.
- **FR-010**: 시스템은 라벨링 진행률(라벨/전체)을 표시해야 한다.
- **FR-011**(P4): 시스템은 작업 유형에 맞는 라벨링 방식을 제공해야 한다(검출=박스, 세그=마스크, 회귀=스칼라값, 이상=정상/이상 토글).

**학습 — 브라우저(빠른 경로)**
- **FR-012**: 사용자는 별도 설치 없이 브라우저에서 학습을 시작할 수 있어야 한다.
- **FR-013**: 시스템은 데이터를 학습/검증으로 자동 분할해야 한다(기본 비율 적용, 분할 정보 노출).
- **FR-014**: 시스템은 학습 진행 중 에폭별 정확도·손실을 실시간으로 표시해야 한다.
- **FR-015**: 학습 차단 조건(라벨 없음, 클래스당 표본 부족 등)을 사전에 검사하고 안내해야 한다.

**학습 — 외부 GPU(본격 경로, P2)**
- **FR-016**(P2): 시스템은 프로젝트의 데이터셋·클래스·설정이 반영된 **실행 가능한 본격 학습 노트북을 생성**해 제공해야 한다.
- **FR-017**(P2): 시스템은 외부 학습이 산출한 **표준 결과(모델 아티팩트 + 평가지표)** 업로드를 받아들여야 한다.
- **FR-018**(P2): 브라우저 학습과 외부 학습의 결과는 **동일한 표준 지표 형식**으로 표시·비교되어야 한다.
- **FR-019**(P2): 업로드된 외부 결과가 프로젝트 설정과 불일치하면 거부하고 불일치 항목을 표시해야 한다.

**결과 & 예측**
- **FR-020**: 시스템은 분류 결과로 혼동행렬, 클래스별 정밀도/재현율, 학습/검증 곡선, 오분류 이미지 갤러리를 표시해야 한다.
- **FR-021**: 사용자는 학습된 모델로 새 이미지를 예측하고 예측 결과와 신뢰도(top-k)를 확인할 수 있어야 한다.
- **FR-022**: 사용자는 학습된 모델과 결과(지표 등)를 내보낼 수 있어야 한다.
- **FR-023**(P4): 시스템은 각 작업 유형에 맞는 평가지표를 표시해야 한다(분류=정확도/F1, 검출=mAP, 세그=IoU, 회귀=RMSE/MAE, 이상=AUROC).

**신뢰성(P5)**
- **FR-024**(P5): 시스템은 결과에 재현에 필요한 정보(데이터 분할, 하이퍼파라미터, 데이터셋 식별)를 기록한 리포트를 제공해야 한다.
- **FR-025**(P5): 시스템은 명백한 데이터 누수(학습/검증 표본 중복 등)를 감지해 경고해야 한다.

**플랫폼/접근**
- **FR-026**: 핵심 동선(프로젝트 생성→예측)은 코드 작성과 로컬 설치 없이 웹 브라우저만으로 완료 가능해야 한다.
- **FR-027**: v0는 로그인·서버 백엔드 없이 동작해야 하며, 프로젝트·이미지·라벨·결과는 **브라우저 로컬에 저장**되어 같은 브라우저에서 세션 간 유지되어야 한다(클라우드 저장·기기 간 공유는 이후 단계).
- **FR-028**: 시스템은 데모 데이터의 출처·라이선스를 표기해야 한다.
- **FR-029**: 시스템은 브라우저 학습(빠른 경로)의 권장 데이터 상한(**총 약 1,500장, 클래스당 300~500장**)을 적용하고, 이를 초과하면 외부 GPU 본격 학습(P2)으로 안내해야 한다.
- **FR-030**: 브라우저 로컬 저장 용량 한도에 근접/초과하면 시스템은 사용자에게 안내하고 정리 또는 내보내기 경로를 제시해야 한다.
- **FR-031**: 시스템의 사용자 인터페이스는 v0에서 **한국어**를 제공해야 한다(영어는 이후 단계).

### Key Entities *(include if feature involves data)*

- **Project(프로젝트)**: 사용자의 작업 단위. 작업 유형, 클래스 정의, 학습 설정을 보유.
- **Image(이미지)**: 업로드/데모 이미지. 크기 및 (천문일 경우) 좌표계 등 메타 보유.
- **Label(라벨)**: 이미지에 부여된 정답. 작업 유형별 형태(클래스/박스/마스크/스칼라/정상·이상)와 출처(수동/폴더/데모) 보유.
- **Run(학습 실행)**: 1회 학습. 실행 위치(브라우저/외부 GPU), 백본, 하이퍼파라미터, 상태 보유.
- **Result(결과)**: 실행 산출물. 표준 평가지표와 모델 아티팩트 참조 보유. 브라우저/외부 실행이 동일 형식 공유.
- **Task Type(작업 유형)**: 분류/검출/세그/회귀/이상. 입력·라벨링·출력·지표·노트북 생성 규칙을 정의하는 추상화 단위.
- **User(사용자, 선택)**: v0는 로그인 없이 동작하며, 모든 데이터는 사용자의 브라우저 로컬에 보존된다(서버 백엔드·계정 없음). 계정/공유는 이후 단계.

**저장 모델(v0)**: 위 엔티티는 모두 **브라우저 로컬**에 저장되며 서버 데이터베이스를 전제하지 않는다. 데모 데이터와 외부 GPU 결과 업로드 처리도 브라우저 로컬 보존을 기준으로 한다.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ML 비전공 사용자가 도움말 없이 **15분 이내**에 데모 데이터로 분류기 학습·예측까지 완주할 수 있다.
- **SC-002**: 신규 사용자 사용성 테스트에서 위 과제 **성공률 ≥ 70%**.
- **SC-003**: 데모 분류 과제(시각적으로 뚜렷한 3클래스)에서 검증 정확도 **≥ 0.80**.
- **SC-004**: 핵심 동선 전체를 **코드 0줄·설치 0**으로 완료할 수 있다.
- **SC-005**: 핵심 사용 경험을 **사용자 비용 0원**(무료 한도 내)으로 제공한다.
- **SC-006**: 브라우저 학습과 외부 GPU 학습 결과가 **동일한 지표 형식**으로 표시되어 사용자가 추가 변환 없이 비교할 수 있다.
- **SC-007**: 분류로 구축한 흐름에 **최소 1개의 추가 작업 유형**(예: 회귀)을 핵심 동선 변경 없이 추가해 끝까지 수행할 수 있다(H3 검증).
- **SC-008**: 업로드/입력 오류가 발생해도 **전체 동선이 중단되지 않고** 개별 항목 단위로 처리된다.
- **SC-009**: 브라우저 학습은 **총 ~1,500장(클래스당 300~500)** 규모를 일반적인 학생용 기기에서 끊김 없이 처리하고, 이 상한을 넘으면 외부 GPU 경로를 안내한다.
- **SC-010**: v0의 모든 데이터는 **재방문(같은 브라우저) 시 유지**되며, 서버/계정 없이도 진행 중 작업을 이어갈 수 있다.

## Assumptions

- 대상 사용자는 천문/물리 학부 고학년~대학원생이며 ML 비전공이다. 현업 프로덕션·대규모 서베이 운영·SOTA 성능은 v1 범위 밖.
- v0 핵심 데모 데이터는 공개 갤럭시 형태 분류 데이터(예: Galaxy10 계열)의 **시각적으로 뚜렷한 3클래스 서브셋**을 사용한다(자세한 후보: [docs/datasets.md](../../docs/datasets.md)).
- 무료 운영을 전제로, 본격 학습은 **사용자 소유의 외부 무료 GPU 환경**에서 수행하고 결과를 업로드하는 모델을 채택한다(자세한 흐름: [docs/compute-hybrid.md](../../docs/compute-hybrid.md)).
- v0는 로그인·서버 백엔드 없이 동작하고 데이터는 브라우저 로컬에 저장한다. 사용자 계정·클라우드 저장·기기 간 공유는 이후 단계에서 도입한다.
- v0 UI 언어는 한국어이며 영어 등 추가 언어는 이후 단계에서 도입한다.
- 데모/사용자 데이터는 공개 또는 사용자 소유 데이터를 가정하며, 라이선스/출처 표기를 제공한다.
- 작업 추상화는 처음부터 일반형으로 설계하되 v0에서는 분류만 채운다(나머지는 인터페이스만 유지: [docs/task-interface.md](../../docs/task-interface.md)).
- 안정적 인터넷 연결과 최신 데스크톱/모바일 브라우저 환경을 가정한다.
