> 천문·우주과학 이미지에서 **머신러닝을 몰라도** 원하는 결과(분류·검출·물리량·이상)를 얻게 해주는 노코드 연구툴.
> 대상: 학부생 ~ 대학원생 (현업에는 애매하더라도 교육·초기연구에 충분).

작성일: 2026-06-25 · 상태: v0 구상 · 코드명: AstroML Studio (변경 가능)

## 0. 한 페이지 캔버스

| 항목 | 내용 |
|---|---|
| **문제** | 천문 이미지 ML은 작업마다 비슷한 파이프라인을 매번 코드로 다시 짠다. 학생은 ML/엔지니어링 장벽 때문에 "이미지에서 원하는 답"에 도달하지 못한다. |
| **사용자** | 천문/물리 학부 고학년·대학원생, 동아리·수업·초기 연구. ML 비전공자. |
| **핵심 가치** | "코드 0줄로 5분 안에 내 천문 이미지로 ML 결과를 얻는다." + 무료. |
| **핵심 통찰** | 천문 이미지 ML의 80%는 정형 파이프라인으로 수렴(전처리→라벨→학습→평가). 차별점은 **천문 도메인(FITS·전처리·엄밀성)** 에 있다. |
| **해결책** | ① 작업 플러그인 추상화(분류/검출/회귀/이상) ② 하이브리드 컴퓨팅(브라우저 전이학습 + Colab 자동 노트북) |
| **차별점** | 범용 노코드(Roboflow 등)엔 없는 **천문 도메인 인식**; astropy류엔 없는 **노코드 UX**. 이 교집합이 빈칸. |
| **무료 구조** | Vercel(프론트) + Supabase(데이터) + 브라우저 ML + 사용자 무료 GPU(Colab) → 운영비 ≈ 0 |
| **검증 지표** | "비전공 학생이 도움 없이 v0로 분류기 완성 (성공률·소요시간)" |
| **최대 위험** | 무료 GPU 학습 동선의 마찰 / 작업 추상화가 4작업을 정말 감당하는가 |

## 1. 문제 정의 (Problem)

- 천문/우주과학 연구에서 이미지 기반 ML은 흔하지만, **작업마다 전처리·모델·평가 코드를 매번 새로 작성**한다.
- 실제로 연구자가 원하는 건 "ML 모델 만들기"가 아니라 **"이미지에서 내가 알고 싶은 것을 얻기"** 다. 모델은 수단일 뿐.
- 그런데 비전공 학생에게는 ML 프레임워크·데이터 파이프라인·GPU 환경 자체가 장벽이라, 정작 과학적 질문에 도달하지 못한다.

**가정 검증 결과 (이 프로젝트의 출발점):**
- ✅ *맞음(프레임워크 수준):* 대부분 "이미지 → 지도학습"이라 백본·파이프라인이 공유된다.
- ⚠️ *부분만 맞음(가치의 위치):* "얻고 싶은 것"은 분류/검출/회귀/이상탐지로 갈래가 많고, 출력 헤드·손실·지표가 다르다. 그리고 진짜 장벽은 **천문 데이터 특수성(FITS·WCS·PSF·노이즈·보정)** 과 **과학적 엄밀성(불확실성·재현성)** 이다.
- → 결론: "범용 이미지 ML 노코드"는 이미 존재. **빈칸은 "천문 도메인 인식 + 노코드 + 학생용".** 여기를 노린다.

## 2. 대상 사용자 (Users)

| 페르소나 | 상황 | 니즈 | 성공 모습 |
|---|---|---|---|
| **학부 고학년** | 수업 과제/동아리 프로젝트 | 코딩 없이 결과·시각화 | 갤럭시 형태 분류기를 한 시간 안에 만들어 발표 |
| **초기 대학원생** | 첫 연구 데이터 탐색 | 빠른 베이스라인 + 진짜 GPU 학습 | 브라우저로 감 잡고 Colab으로 본 학습 |
| **교육자** | ML 입문 수업 | 설치 없는 실습 도구 | 학생이 링크만으로 실습 |

비대상(현 단계): 대규모 서베이 운영, 프로덕션 추론 파이프라인, 논문급 SOTA 성능.

## 3. 핵심 가설 (Hypotheses) — 검증 가능하게

1. **H1 (가치):** 비전공 학생이 도움 없이 v0로 이미지 분류기를 완성할 수 있다. → *지표: 과제 성공률 ≥ 70%, 중앙 소요시간 ≤ 15분.*
2. **H2 (기술):** 브라우저 전이학습(헤드만)이 학생 데모 수준에서 "쓸 만한" 정확도를 낸다. → *지표: 갤럭시 형태 2~3클래스에서 정확도 ≥ 0.8.*
3. **H3 (확장성):** 분류로 만든 작업 플러그인 추상화가 검출/회귀/이상탐지로 코드 대수술 없이 확장된다. → *지표: 새 작업 추가 시 공통 코어 변경 최소.*
4. **H4 (비용):** 운영비 0(무료 티어)로 의미 있는 사용 경험을 제공한다. → *지표: 무료 한도 내 데모 완주 가능.*

## 5. 접근 방법 (Approach)

### 5.1 작업 플러그인 추상화 (가장 비싼 결정)
모든 작업이 공통 인터페이스를 구현한다. v0는 분류만 채우고 나머지는 인터페이스만 비워둔다 → 확장이 "새 모델 만들기"가 아니라 "빈칸 채우기".

```
Task 인터페이스
├── 입력 스키마        이미지 / FITS / 이미지+메타
├── 라벨링 UI          클래스버튼 / 박스 / 마스크 / 스칼라값
├── 출력 헤드          softmax / bbox / mask / regression
├── 손실·평가지표      accuracy·F1 / mAP / IoU / RMSE·MAE
└── Colab 노트북 템플릿  작업별 학습 코드 생성기
```

지원 작업(비전):
1. **분류** (v0) — 갤럭시 형태 등
2. **검출/세그멘테이션** — 소스 위치·영역
3. **물리량 회귀** — photometric redshift, 광도/질량 등
4. **이상탐지** — transient·변광 등

→ 구체 인터페이스(7개 슬롯)와 작업별 매핑: [작업 플러그인 인터페이스](docs/task-interface.md)

### 5.2 하이브리드 컴퓨팅 (무료의 핵심)
무료로 GPU를 상시 백엔드에 띄우는 건 사실상 불가능하므로 우회한다.

```
업로드 → (브라우저) 전처리·라벨링 → 두 갈래
  ├─ [빠른 경로] 브라우저 전이학습(TF.js, 헤드만) → 즉시 정확도/혼동행렬
  └─ [본격 경로] Colab 노트북 자동생성 → 사용자가 무료 GPU로 실행
                 → 학습된 모델·지표 업로드 → 사이트에서 시각화·비교
```
같은 데이터셋·라벨을 두 경로가 공유 → "브라우저로 감 잡고 → Colab으로 제대로" 동선.

## 7. 데이터 모델 (초안)

- **Project** — 사용자의 작업 단위 (작업 유형, 클래스 정의, 설정)
- **Image** — 업로드 이미지/FITS, 메타(WCS 등)
- **Label** — 작업별 라벨(클래스/박스/마스크/스칼라)
- **Run** — 학습 실행(브라우저 or Colab), 하이퍼파라미터
- **Result** — 모델 아티팩트 참조 + 평가지표 + 시각화

## 8. 검증 계획 (Validation)

| 가설 | 방법 | 성공 기준 |
|---|---|---|
| H1 | 비전공 학생 N명 사용성 테스트 | 성공률 ≥ 70%, 중앙시간 ≤ 15분 |
| H2 | 갤럭시 형태 데모셋 벤치 | 정확도 ≥ 0.8 |
| H3 | 두 번째 작업(검출) 추가 시 코어 변경량 측정 | 코어 대수술 없음 |
| H4 | 무료 한도 내 데모 완주 | 한도 초과 없이 완료 |

## 9. 위험 & 한계 (Risks)

- **R1 무료 GPU 마찰:** Colab 왕복(노트북 실행→결과 업로드)이 번거로울 수 있음 → 브라우저 빠른 경로로 첫 가치를 먼저 제공해 완화.
- **R2 추상화 과적합:** 분류 기준으로 만든 인터페이스가 검출/회귀에 안 맞을 위험 → v0에서 인터페이스를 "분류 전용"이 아니라 일반형으로 설계, 두 번째 작업으로 조기 검증.
- **R3 과학적 신뢰성:** 학생툴이라도 데이터 누수·불확실성 무시 위험 → v3에서 재현성·불확실성 리포트.
- **R4 브라우저 성능 한계:** 대규모 데이터 학습 불가 → 본격 경로(Colab)로 분리.

## 11. 비전 vs 지금 (스코프 경계)

- **지금(v0) 깊게:** 분류 작업 end-to-end, 일반형 작업 인터페이스, 하이브리드 흐름.
- **비전으로만(한 줄씩):** 검출/회귀/이상탐지 상세, FITS 파이프라인 세부, 불확실성 리포트.
- → Spec Kit으로 정식화할 때도 이 경계를 유지(상세 contracts는 v0 + 인터페이스에만).

## 12. 용어 (Glossary)

- **FITS** — 천문 표준 이미지/데이터 포맷.
- **WCS** — 픽셀↔하늘 좌표 변환.
- **photometric redshift (photo-z)** — 이미지/측광에서 추정한 적색편이.
- **전이학습** — 사전학습 백본을 고정하고 작은 헤드만 새로 학습.
- **transient** — 시간에 따라 나타났다 사라지는 천체(초신성 등).
